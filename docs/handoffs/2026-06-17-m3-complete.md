## What exists now (and works)

End-to-end log capture + live metrics over the existing daemon/gRPC stack. Quick proof
(short `/tmp` base for the macOS socket-path limit):
```bash
go build -o marshal ./cmd/marshal
export XDG_DATA_HOME=/tmp/m3smoke && rm -rf "$XDG_DATA_HOME"
printf 'apps:\n  - name: clock\n    cmd: sh\n    args: ["-c","i=0; while true; do echo tick-$i; i=$((i+1)); sleep 1; done"]\n    instances: 2\n' > /tmp/m3.yaml
./marshal start /tmp/m3.yaml   # 2 procs
sleep 6
./marshal list                # CPU/MEM/UPTIME columns populated for online procs
./marshal logs clock -n 5      # recent tick lines, tagged clock#0 / clock#1, time-ordered
./marshal logs clock -f        # live follow (Ctrl-C to stop)
./marshal kill
```

### Architecture (M3 additions)
```
  process stdout/stderr
     │  (proc.Spec.Stdout/Stderr — nil ⇒ inherit terminal, so `marshal run` is unchanged)
     ▼
  manager (WithLogs(LogProvider)) ── per-instance ──▶ logs.Sink
                                                       ├─ tee raw bytes ─▶ lumberjack file (10MB×5)
                                                       └─ split lines ─▶ ring (~1000) + subscriber fanout
  daemon owns logs.Registry (label→Sink) and metrics.Sampler (gopsutil, 5s)
     ├─ Logs RPC: Describe→ResolveLabeled→mergeBackfill (ring, time-ordered) → [follow] Subscribe fan-in
     └─ List/Describe: merges sampler.Get(label) cpu/mem into each ProcInfo
  CLI: `marshal logs` streams + prints `name#inst | text` (stderr → client stderr);
       `list`/`describe` gain CPU/MEM columns.
```

### Packages (new in M3)
- `internal/logs` — `Sink` (`sink.go`: tee→lumberjack + fixed `ring` + drop-on-full
  subscriber fanout; `Writer`/`Backfill`/`Subscribe`/`Close`) and `Registry`
  (`registry.go`: one Sink per `name#idx` label; `For` create-or-reuse, `WriterPair`,
  `Remove`, `ResolveLabeled`). All state guarded by a single mutex; close/send serialized
  so no send-on-closed-channel.
- `internal/metrics` — `Sampler` (`sampler.go`): every 5s sums CPU%/RSS over each online
  instance's **process group** (`groupPids` = main pid + recursive `Children()`); retains
  `*process.Process` handles across ticks for gopsutil `Percent(0)` deltas; prunes dead
  pids. `NewSampler(interval)`, `Run(ctx, snapshot)`, `Get(label)`, `SampleOnce` (sync,
  for tests/daemon).

## Key decisions / non-obvious things

- **Daemon owns logs+metrics; manager stays lifecycle-only.** Keeps the manager testable
  and import-clean and matches the eventual multi-host agent direction.
- **`proc.Spec` nil writers ⇒ inherit terminal**, so M1 `marshal run` is byte-for-byte
  unchanged (foreground builds the manager *without* `WithLogs`).
- **Sinks are reused across restarts** (`Registry.For` is create-or-return), so an explicit
  `restart` keeps the same files and ring history; only `Delete` disposes (`Remove`).
- **Ring-only backfill** (`-n N` served from memory; deep history beyond the ring is
  sub-project #2). Multi-instance `logs` output is **merged + tag-prefixed** (`name#inst`),
  stderr routed to the client's stderr.
- **Metrics = whole process group** (shell-wrapped apps report real usage), via recursive
  `groupPids`; gopsutil CPU% is a delta, hence the retained-handle cache + priming sample.
- **`Logs` RPC**: `Unavailable` if logs not configured, `NotFound` for an unknown selector
  (via `Describe`); follow uses one fan-in goroutine per sink and exits on
  `stream.Context().Done()` (gRPC cancels the stream ctx when the handler returns).
- **No proto contract change** — M2 already reserved `Logs`/`cpu`/`mem`.
- Security preserved: socket `0600`, state + logs dirs `0700`.

## Deferred / known issues (address later)

- **Follow backfill→subscribe gap** (`internal/daemon/logs.go`): backfill is snapshotted
  *before* `Subscribe`, so a `-f` client can miss a line or two emitted in that window
  under high output (no duplication). Fine for a tail; if the fleet server needs
  no-dropped-lines, switch to subscribe-then-backfill-then-dedup-by-timestamp.
- **No max-line cap in `logs.Sink`**: a process emitting a very long newline-less chunk
  grows the pending-line buffer unbounded until a `\n` arrives. Inherent to line-buffering;
  add a flush/truncate guard before facing untrusted output.
- **`logs.Sink` file I/O under the sink mutex**: a stdout rotation briefly blocks the stderr
  writer / readers. Latency-only, not a hazard; move the file tee outside the lock if
  contention ever matters.
- **`groupPids` walks `Children()` every 5s tick** — fine single-host; revisit cost for
  deep/fork-heavy trees on the future fleet agent.
- **First CPU% reading** for a freshly-spawned pid can be slightly off (gopsutil baseline);
  the priming sample mitigates steady-state. Cosmetic.
- Carried from M2: auto-spawn has no cleanup on `waitReady` timeout; `Resurrect` swallows
  per-app `Add` errors; untrusted `dump.json` trust boundary; Windows still deferred.
