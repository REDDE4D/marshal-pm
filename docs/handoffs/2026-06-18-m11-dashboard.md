## What was built

### `internal/server/auth.go` (Task 1) — dashboard credentials

`dashboardUser{PBKDF2, Salt, Iter}` records appended to the existing `auth.json` `"users"`
map alongside the fleet tokens. `SetDashboardUser` / `VerifyDashboardUser` / `HasDashboardUser`
on `*AuthStore`. PBKDF2-HMAC-SHA256 at 600 000 iterations, 16 B salt, 32 B key; `subtle.ConstantTimeCompare`
verify; atomic save + rollback. Exported wrappers: `SetDashboardPassword`, `HasDashboardUserDir`.

### `internal/dashboard/session.go` (Task 2) — server-side session store

`sessionStore{ttl, now, mu, m}`. `create` mints a 32 B crypto/rand base64-URL-encoded token;
`validate` checks expiry (constant-time delete+false on miss); `delete`; `sweep` (periodic
eviction). Injectable clock (`nil` → `time.Now`).

### `internal/dashboard/fleet.go` (Task 3) — fleet view mapping

`FleetLister` interface (`List() []*pb.AgentState`), satisfied by `*server.Registry`.
`fleetView(lister)` maps proto getters to `agentView{name, connected, last_seen_unix, procs}`
and `procView{name, state, pid, uptime_ms, restarts, cpu, mem}`.

### `internal/dashboard/handlers.go`, `embed.go`, `server.go` (Task 4) — HTTP handler + TLS serve

`NewHandler(auth, sessions, fleet)` wires a Go 1.22 pattern mux:
- `POST /api/login` — verifies credentials, mints session, sets `marshal_session` cookie (HttpOnly/Secure/SameSiteStrict).
- `POST /api/logout` — always clears cookie (intentionally unguarded; clears even expired sessions).
- `GET /api/session` — returns `{"user":"..."}` (guarded by `requireSession` middleware → 401).
- `GET /api/fleet` — guarded; returns JSON-encoded fleet view.
- Everything else — SPA: real files served from the embedded `fs.Sub(dist, "dist")`; any other path falls back to `index.html`.

`Serve(ctx, addr, fleet, auth, cert)` runs `http.ServeTLS` + `Shutdown(5 s)` on ctx cancel;
ignores `http.ErrServerClosed`.

`dist/index.html` is committed as the embedded asset seed (replaced by `make ui`).

### `cmd/marshal/server_dashboard.go` (Task 5) — `marshal server passwd`

`--user` (default `admin`) / `--data-dir`. TTY path: `term.ReadPassword` × 2 with
confirm-match + empty reject. Non-TTY path: `bufio.Scanner` single line, `TrimRight \r\n`,
empty reject. Registered as `server passwd` subcommand.

### `internal/server/server.go` (Task 6) — `--http-listen` wiring

`ServeDir(..., httpAddr string, opts...)` creates `*Registry` once; when `httpAddr != ""`
it logs a hint if no dashboard user is set, then starts `go dashboard.Serve(ctx, httpAddr, reg, auth, cert)`.
The same `reg` is passed to the gRPC `Serve` — dashboard reads live fleet state. Empty
`httpAddr` preserves identical prior behavior.

`cmd/marshal/server.go`: `--http-listen` flag (default `""`, dashboard disabled unless
explicitly set).

Integration test `TestServeDirStartsDashboard` in `internal/server/dashboard_serve_test.go`:
verifies 401 without cookie, then 200 login + session cookie, then 200 authenticated
`/api/fleet` — via an `InsecureSkipVerify` TLS client with a 100 × 20 ms retry loop on
the listen port.

### `web/` + `internal/dashboard/dist/` (Task 7) — React/Vite SPA

`web/` tree: Vite + React 19 + TypeScript. Key files:
- `src/api.ts` — `login({User, Pass})`, `getFleet()` (throws on 401).
- `src/Login.tsx` — login form; calls `api.login`, invokes `onLogin` on success.
- `src/Fleet.tsx` — polls `/api/fleet` every 2 s (cleanup on unmount), renders agent table
  with per-process rows and a human-friendly uptime helper.
- `src/App.tsx` — three-state router: `null` → Loading, `false` → `<Login>`, `true` → `<Fleet>`.

`vite.config.ts`: `outDir: ../internal/dashboard/dist`, `emptyOutDir: true`. Built dist
(hashed `index.html` + `assets/index-*.{js,css}`) is **committed and tracked** — no build
step required to `go build`.

`Makefile` targets: `make ui` (installs npm deps + builds SPA), `make build` (Go binary),
`make test` (race tests).

`.gitignore` fix: `/dist/` (root-only) so the committed `internal/dashboard/dist/` is not
ignored.

## Build / run / test

### One-time SPA rebuild (only needed when changing `web/src/`)

```bash
make ui    # runs: cd web && npm install && npm run build
```

The built dist is already committed; `go build` embeds it without a Node toolchain.

### Build and test Go

```bash
go build -o marshal ./cmd/marshal
go test ./... -race -count=1
gofmt -l .    # must print nothing
go vet ./...
```

### Run the server with dashboard

```bash
# First run: tokens are printed; capture them.
./marshal server --listen :9000 --http-listen :8443

# Set the dashboard password BEFORE or AFTER starting the server.
# If set while server is running, restart the server for it to pick up
# the new credentials (server loads auth.json only on start; in-memory
# AuthStore is not live-reloaded by the passwd command).
printf 'yourpassword\n' | ./marshal server passwd --user admin
```

**Recommended first-run sequence** to avoid the in-memory stale-credentials problem:

```bash
# 1. Start server once to generate and persist the auth tokens (printed to stdout):
./marshal server --listen :9000 --http-listen :8443 &
#    ← copy the "enroll token" and "cert fingerprint" from the output
kill %1   # stop it

# 2. Set the dashboard password in the now-initialized data dir:
printf 'yourpassword\n' | ./marshal server passwd --user admin

# 3. Restart the server — it loads all credentials from auth.json (tokens + user):
./marshal server --listen :9000 --http-listen :8443
```

Then open `https://localhost:8443/` in a browser (accept the self-signed cert warning),
log in with `admin` / your password.

### Enroll an agent

```yaml
# app.yaml
apps:
  - name: ticker
    cmd: /bin/sh
    args: ["-c", "while true; do echo tick; sleep 2; done"]
server:
  address: localhost:9000
  name: dev-1
  token: <ENROLL_TOKEN_FROM_SERVER_STDOUT>
  fingerprint: <FINGERPRINT_FROM_SERVER_STDOUT>
```

```bash
./marshal start app.yaml   # auto-enrolls; persists <state-dir>/fleet-token
```

file but NOT the running server's in-memory store. Restart the server after calling these
commands for the new credentials to take effect. This is a known deferred issue (see below).

## Deferred / known issues

These do not block merge; they are carried forward to later milestones.

### Feature gaps (future milestones)

1. **Metric charts** (M12) — the fleet table shows CPU/mem numbers but no sparklines or
   historical charts; those require the metric-history store already built in M5/M8.
2. **Live log tailing** (M13) — `/api/fleet` returns process state; per-process log streams
   are a separate feature using the log-store API from M6/M8b.
3. **Multi-user accounts** (M13+) — `server passwd` manages a single user per name; there is
   no user list, delete, or role system yet.
4. **Process controls** (M14) — the dashboard is read-only; start/stop/restart buttons require
   wiring to the `FleetControl` gRPC channel.

### Session / auth limitations

5. **In-memory sessions lost on server restart** — the `sessionStore` is RAM-only. All
   logged-in sessions are invalidated when the server restarts; users must log in again.
6. **`server passwd` / `server token --rotate` not hot-reloaded** — these commands write a
   new `AuthStore` to disk but do not signal the running server. Restart required to pick up
   new credentials.
7. **No login rate-limiting** — brute-force against `/api/login` is unconstrained. Add an
   in-memory failure counter or per-IP throttle before exposing the dashboard on a public
   network.

### UX / deployment

8. **Self-signed cert browser warning** — the server auto-generates a self-signed cert.
   Users must click through the browser warning on first visit. Provide `--tls-cert`/`--tls-key`
   flags (already exist, inherited from M10) to substitute an operator-supplied cert for a
   smoother experience.

### Minor code-quality follow-ups (non-blocking, surfaced in task reviews)

- Redundant `ensureDataDir` before `loadOrInitAuth` in `SetDashboardPassword`/`HasDashboardUserDir`
  (double `MkdirAll`).
- Dead nil-guard on `a.data.Users` in `SetDashboardUser` (map is always initialised by
  load paths).

From Task 2 (session.go):
- `TestSessionSweep` doesn't assert a live session survives sweep.
- No test for `newSessionStore(ttl, nil)` clock fallback.

From Task 3 (fleet.go):
- `TestFleetView` doesn't assert `UptimeMs`/`CPU`/`Mem` proc fields (brief-inherited).

From Task 4 (handlers.go):
- SPA probes file via `Open`+`Close` instead of `fs.Stat` (cosmetic).

From Task 5 (server_dashboard.go):
- TTY path checks mismatch before checking empty, so double-empty input yields "passwords do
  not match" instead of "empty password" (cosmetic).

From Task 6 (server/server.go, dashboard_serve_test.go):
- Integration test reserves+closes+rebinds port (TOCTOU race). Clean fix is to pass a
  `net.Listener` to `dashboard.Serve` (interface change, out of scope).
- `resp.Body` not closed in test (fd leak, low risk).
- "dashboard: serving on" is logged before the goroutine actually binds (cosmetic).

From Task 7 (Fleet.tsx, api.ts):
- `Fleet.tsx` `err` state never set to non-empty (dead code path).
- `api.ts` `getSession` casts `j.user as string` without a guard.
- Two npm dev-dep audit advisories (toolchain only, not present in the built artifact).
