every fleet RPC requires TLS and authentication, agents have per-agent identity, and

## What was built (the four mechanisms)

1. **TLS everywhere + fingerprint pinning.** The server auto-generates a self-signed
   cert+key (`<data-dir>/cert.pem|key.pem`, 0600) on first run and prints the SHA-256
   fingerprint; `--tls-cert`/`--tls-key` override with operator-supplied certs. Agents/CLI
   connect over TLS and pin the fingerprint (`fleetauth.ClientTLS`: `InsecureSkipVerify` +
   a constant-time `VerifyPeerCertificate` against the pinned hash), or trust a CA file.
   Trust must be explicit (`fingerprint` XOR `ca`) — no implicit system-roots, no plaintext
   fallback, no `--insecure`.
2. **Agent auto-enrollment + per-agent identity.** The server holds one hashed **enroll
   token**. An agent's first `Connect` presents `marshal-enroll` + a requested name; the
   server registers it, mints a per-agent token, returns it in `HelloAck.agent_token`, and
   the agent persists it to `<state-dir>/fleet-token` (0600). Subsequent connects present
   `marshal-token`; the server trusts the **authenticated** name, not the self-asserted
   `Hello.agent_name`. Revoke with `marshal server agent rm <name>`.
3. **Single admin token for operators.** The server generates a hashed **admin token**; the
   CLI sends it (`marshal-token` metadata) on every operator RPC.
4. **Server-side authorization.** gRPC interceptors run before any handler:
   `unaryAuth` requires the admin token on operator RPCs; `streamAuth` requires a valid
   per-agent token or enroll token on `Connect`. Missing → `Unauthenticated`, invalid →
   `PermissionDenied`. `FleetControl` cannot reach the broker without passing the gate.

## Architecture (where things live)

- **`internal/fleetauth/`** (new, pure): `tls.go` (`Fingerprint`, `ClientTLS`), `token.go`
  (`GenerateToken`/`HashToken`/`VerifyToken`, constant-time). Shared by agent, CLI, server.
- **`internal/server/tls.go`**: `LoadOrCreateCert(dir, certPath, keyPath)` — generate or load;
  ensures the cert dir exists (see fix below).
- **`internal/server/auth.go`**: `AuthStore` over `auth.json` (0600, atomic tmp+rename) —
  hashed enroll/admin tokens + per-agent registry (`name → {token_hash, enrolled_at}`).
  `LoadOrInitAuth` (mints+persists+returns plaintext on first run), `verifyAdmin/verifyEnroll`,
  `enrollAgent`/`authAgent`/`removeAgent`/`listAgents`/`rotate`, with rollback on save
  failure. Exported wrappers: `FingerprintForDir`, `RotateToken`, `ListAgents`, `RemoveAgent`,
  `InitAuthPrint`.
- **`internal/server/interceptor.go`**: `unaryAuth`/`streamAuth` + `wrappedStream` injecting
  `authedAgentName`/`isEnrolling` into the Connect context.
- **`internal/server/server.go`**: `NewServer(reg, ss, ls, auth)`, `Serve(..., cert, auth)`
  (installs both interceptors; **rejects a nil AuthStore**), `ServeDir` loads cert+auth;
  `Connect` honors enroll vs authenticated identity.
- **`internal/fleet/client.go`**: `WithTLS(cfg)`, `WithAuth(token, enrollToken, persist)`;
  `connectOnce` attaches metadata, persists the minted token from the ack, and switches to
  it on reconnect.
- **`internal/daemon/server.go`**: loads `fleet-token`; if no token and no enroll token →
  logs `fleet: disabled — no token and not enrolled` and skips; else wires `WithTLS`+`WithAuth`.
- **`internal/config/config.go`**: `ServerConfig` += `Token`, `Fingerprint`, `CA`; validates
  exactly one trust source.
- **`internal/store/store.go`**: `FleetTokenPath`/`LoadFleetToken`/`SaveFleetToken`.
- **`cmd/marshal/server.go`** + **`server_auth.go`**: `--tls-cert`/`--tls-key`; first-run
  secret printing; `server fingerprint | token --rotate enroll|admin | agent ls|rm`.
- **`cmd/marshal/fleet.go`**: `resolveServerAuth` (flag → env → `~/.config/marshal/cli.yaml`
  → default), `dialFleet`, `authCtx`; `--token`/`--fingerprint` on every fleet subcommand.
- **`proto/marshal/v1/fleet.proto`**: `HelloAck.agent_token = 3` (additive, wire-compatible).

## Build / run / test

```bash
go build -o marshal ./cmd/marshal
go test ./... -race -count=1
gofmt -l .   # must print nothing
go vet ./...

# Server (first run prints enroll token, admin token, fingerprint — capture them)
./marshal server --listen :9000

# Agent app.yaml:
#   server: {address: localhost:9000, name: dev-1, token: <ENROLL>, fingerprint: <FP>}
./marshal start app.yaml          # auto-enrolls; persists <state-dir>/fleet-token

# Operator (admin token + fingerprint required on every command)
./marshal fleet ps      --server localhost:9000 --token <ADMIN> --fingerprint <FP>
./marshal fleet restart dev-1 ticker --server localhost:9000 --token <ADMIN> --fingerprint <FP>
./marshal server agent ls
./marshal server fingerprint
./marshal server token --rotate admin
```

CLI defaults can also come from env (`MARSHAL_SERVER`/`MARSHAL_TOKEN`/`MARSHAL_FINGERPRINT`)
or `~/.config/marshal/cli.yaml` (`server:`/`token:`/`fingerprint:`).

## Smoke-test proof (2026-06-18, `XDG_DATA_HOME=/tmp/m10smoke`, server `:9100`)

- Fresh `marshal server` printed enroll/admin tokens + fingerprint; wrote `auth.json`,
  `cert.pem`, `key.pem` (all 0600).
- `marshal start app.yaml` (ticker) → agent **auto-enrolled**: `fleet-token` persisted;
  `auth.json` gained `dev-1`; `fleet ps` (admin token + fp) showed `ticker` online.
- `fleet restart dev-1 ticker` (admin token + fp) → routed over the enrolled stream, agent
  executed it.
- **Negative checks all rejected:** no token → `missing admin token`; no fingerprint →
  `no TLS trust source`; wrong token → `invalid admin token`; wrong fingerprint → TLS
  handshake `... does not match pinned ...`.
- `server agent ls` → `dev-1 enrolled ...`; `server fingerprint` matched the served cert.

A **fresh-start bug was found and fixed during smoke** (`dee2c2a`): `LoadOrCreateCert` wrote
the cert before the data dir existed (unit tests used `t.TempDir()`, which masked it). It now
`MkdirAll`s the cert dir; regression test `TestLoadOrCreateCertCreatesMissingDir` added.

### Minor code-quality follow-ups surfaced in task reviews (non-blocking, for final-review triage)

- `internal/fleetauth/tls.go`: wrong-length pinned fingerprint yields a generic "does not
  match" message (could say "wrong length"); a few test-coverage nits (no both-trust-sources
  test; token length test counts base64 chars not raw bytes).
- `internal/fleet/client.go`: `connectOnce` silently proceeds with no metadata when `WithAuth`
  is never called (`persistTok == nil`) — fails closed at the server interceptor, but deviates
  from a client-side fail-fast; two stub-server unit tests rely on this. Cleanup = migrate
  those two tests to `WithAuth` and make `connectOnce` error unconditionally without auth.
- `internal/server/auth.go`: `EnrollAgent` is exported solely for a `cmd/marshal` test
  (widens a security type's mutation surface) — cleaner to move the `agent ls`/`rm` tests
  into the `server` package and unexport it; redundant double-`MkdirAll` in
  `RotateToken`/`ListAgents`/`RemoveAgent`.
- `cmd/marshal/fleet.go`: read commands (`fleet ps/metrics/logs`) print cobra usage on a
  server-side error (no `SilenceUsage`), unlike the control commands.
- `cmd/marshal/fleet_test.go`: a redundant pre-dial in `TestFleetRestartSendsControlOp`.
