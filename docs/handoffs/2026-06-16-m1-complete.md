## What exists (and works)

`marshal run <marshal.yaml>` — a foreground process supervisor:
- `internal/config` — parses `marshal.yaml` (apps: name, cmd, args, cwd, instances, env,
  restart mode, max_restarts, kill_timeout), applies defaults, validates.
- `internal/proc` — spawns one OS process; **each process is its own process group**
  (`Setpgid`), and Signal/Kill target the whole group so child trees (e.g. shell wrappers)
  are stopped completely with no orphans. Injects `MARSHAL_INSTANCE_ID`.
- `internal/supervisor` — `Instance` restart state machine (states: starting/online/
  stopping/stopped/restarting/errored), exponential backoff (100ms→15s), min-uptime guard
  (1s), max-restarts cap (→ errored), graceful stop (SIGTERM → kill_timeout → SIGKILL).
  Plus a pure `Backoff` helper. Concurrency is race-clean (verified with `-race`).
- `internal/manager` — fans each app into N fork-mode instances and supervises them all
  concurrently; exposes labeled snapshots.
- `cmd/marshal` — cobra CLI; `run` command with SIGINT/SIGTERM-driven graceful shutdown
  and periodic status lines.

Quick check it works:
```bash
go build -o marshal ./cmd/marshal
printf 'apps:\n  - name: clock\n    cmd: sh\n    args: ["-c","while true; do date; sleep 1; done"]\n    instances: 2\n' > /tmp/marshal.yaml
./marshal run /tmp/marshal.yaml   # Ctrl-C to stop cleanly
```

## Key decisions / non-obvious things this session

- **Process groups were added beyond the original M1 plan** as a real correctness fix: the
  end-to-end test exposed that signaling only the direct child leaks grandchildren (and an
  orphaned grandchild holding the stdout pipe hangs `exec.Cmd.Wait`). This is now handled
  in `internal/proc`.
- The CLI e2e test (`cmd/marshal/run_test.go`) **polls for an app-readiness token before
  interrupting** and uses a mutex-guarded buffer — do not "simplify" it back to a fixed
  `time.Sleep`; that races process startup under load.
- `Snapshot().Pid` is cleared (0) when a process is not running, so the status display
  never shows a stale dead PID.

## Deferred (intentionally out of M1 scope — address in later milestones)

- `max_restarts: 0` currently means "use default 16", not "unlimited". If unlimited is
  wanted, switch `config.App.MaxRestarts` to `*int` (a breaking change — decide before
  consumers depend on it).
- Add a `supervisor.Policy.Validate()` (e.g. reject zero kill_timeout / max_backoff <
  base_backoff).
- Process-group supervision is Unix-only (`Setpgid`); Windows needs separate handling.
- Per-test timeout guards on the timing-based supervisor tests (low priority).

- **M1 — foreground supervisor ✅ DONE** (plan:

- **M3 — logs to rotated files + live metrics** (gopsutil CPU/mem sampling).
- **M4 — boot startup** (systemd unit / launchd plist generation).
